# Story 1.1: Scaffolding e Gestão de Gastos

## Status

Done

## Story

**As a** usuário do Kekel Finance,
**I want** um projeto React configurado e funcional com CRUD completo de gastos por categoria,
**so that** eu possa começar a registrar e visualizar meus gastos mensais imediatamente.

## Acceptance Criteria

1. Projeto inicializado com Vite + React 18 + TypeScript (strict mode) + Tailwind CSS v3
2. `useFinanceStore` criado com Zustand + persist middleware (localStorage)
3. Tipos definidos: `Expense { id, amount, description, categoryId, date, createdAt }` e `Category { id, name, color, isDefault }`
4. Categorias padrão seed ao inicializar o store: Alimentação, Transporte, Moradia, Saúde, Lazer, Educação, Outros
5. Tela de listagem de gastos do mês atual — ordenados por data, com filtro por categoria
6. Formulário de novo gasto: valor (obrigatório), categoria (obrigatório), descrição (opcional), data (padrão = hoje)
7. Edição e exclusão de gasto existente com confirmação de exclusão
8. Total do mês calculado e exibido em destaque na tela de listagem
9. Layout responsivo mobile-first (375px mínimo) com navegação bottom-bar placeholder para as próximas telas
10. `npm run build` sem erros; `tsc --noEmit` sem erros

### Refactoring Performed

- **File**: `kekel-finance/src/store/useFinanceStore.ts` (linha 46)
  - **Change**: Substituído `new Date(e.date)` + `getFullYear()` / `getMonth()` por parsing direto da string `e.date.split('-').map(Number)`
  - **Why**: ISO date-only strings (`"2026-03-01"`) são interpretadas como UTC midnight pela spec ECMAScript. Em fuso horário UTC-3 (Brasil), `new Date("2026-03-01")` vira `2026-02-28T21:00:00` local, fazendo o primeiro dia de cada mês ser filtrado no mês errado.
  - **How**: Parseando `y` e `m` diretamente da string, sem criar um objeto `Date`, eliminamos completamente a dependência de timezone.

- **File**: `kekel-finance/src/components/expenses/ExpenseForm.tsx` (linha 10)
  - **Change**: `today()` reescrita para usar `d.getFullYear()`, `d.getMonth() + 1`, `d.getDate()` em vez de `new Date().toISOString().split('T')[0]`
  - **Why**: `toISOString()` retorna a data em UTC. Após as 21h no Brasil (UTC-3), isso retornaria a data de amanhã como padrão do campo Data.
  - **How**: Construindo a string `YYYY-MM-DD` a partir do objeto `Date` local, garantimos que o padrão reflita a data real do usuário.
